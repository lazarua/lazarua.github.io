1、创建文章

# 创建一个新文章
hexo new [layout] <title>

[layout] ——文章的布局可省略，默认为 post

< title > —— 文章标题

例：hexo new 1、Hexo博客写作

命令会在source/_post文件夹中创建了一个文件，命名为：1、Hexo博客写作.md

2、Front-matter

Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件的变量

打开生成的文件，里面有自定义的参数，如下

---
title: 1、Hexo博客写作-markdwm语法
date: 2021-09-24 18:26:22
tags:
---

​title ——文章名

​date ——建立时间

​tags ——标签

扩展

参数 描述 默认值
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
categories 分类（不适用于分页）

3、编写文章（基于Markdown）

Markdown简介：Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式

（1）标题

# 一级标题
##  二级标题
### 三级标题

（2）代码高亮

高亮的定义：通过字体的颜色或背景色，突出某段内容。分为：行内代码高亮、代码块高亮、（普通文本高亮）

行内代码高亮

语法：

`行内代码高亮`

效果：

行内代码高亮

代码块高亮

语法：

```
代码块高亮
代码块高亮
```

效果：

代码块高亮
代码块高亮

普通文本高亮

CommonMark中并未定义普通文本高亮。如果你使用的markdown编辑器不支持该便捷设置，可通过HTML标签<mark>实现，注意：不同编辑器对<mark>标签的样式实现可能会略有不同：

语法：

<mark>普通文本高亮<mark>

效果：

文本高亮

其他

样式名 效果 Markdown
加粗 文本 *文本* 或 __文本__，用两个*或两个_包围文本
斜体 文本 *文本或_文本_，用一个或一个_包围文本
删除线 文本 ~~文本~~
小号字体 小号字体 <small>小号字体</small>
大号字体 大号字体 <big>大号字体</big>

3、插入图片——Markdown方法

（1）绝对引用：使用 /source/images 文件夹

将图片文件放在 /source/images 文件夹下, 使用 markdown 语法

![代替图片的文字](/images/假装有图.png)

可以实现图片显示。

（2）相对引用（推荐）

在Hexo根目录中*_config.yml* 中配置指令 post_asset_folder: true 这样在每次新建文章的时候就会建立同名文件夹, 直接将相应的文章图片放在文件夹内使用markdown语法访问同名文件夹下图片资源,十分方便。

![a](标题.png)

（3）在线资源

如果我们要的图片现在网络上已经存在了,我们找到相应的地址直接引用就好了。可以只用标准markdown 图片语法引用相关图片

![图片1](http://qzyy987dp.bkt.clouddn.com/%E7%8C%AB%E7%8C%AB.jpeg)

4、插入图片——使用HTML引用（推荐，可控制图片大小）

语法：

<img src="http://qzyy987dp.bkt.clouddn.com/%E7%8C%AB%E7%8C%AB.jpeg" width=256 height=256 />

固定图片大小：width=256,height=256设置的是显示图像的尺寸，src后面接的就是图像对象

还可以使用百分比的方式来给width,height传参

<img src="http://qzyy987dp.bkt.clouddn.com/%E7%8C%AB%E7%8C%AB.jpeg" width="50%" width="50%"/>

其中width和height后面的50%就是根据窗口的大小以一定的比例显示图片。值得一说的是，这种按照百分比显示方法，是以width方向为准，并保持纵横比的。因此用此方法的时候，可以把height设置缺省。如：

<img src="https://alexcld.com/images/pengyuyan.jpg" height="5%" />

5、插入表格

| 标题1  | 标题2 |  标题3 |
| :----- | :---: | -----: |
| 左对齐 | 居中  | 右对齐 |
| 内容1  | 内容2 |  内容3 |
标题1 标题2 标题3
左对齐 居中 右对齐
内容1 内容2 内容3

语法说明：

  1. 不管是哪种方式，第一行为表头，第二行分隔表头和主体部分，第三行开始每一行代表一个表格行；
  2. 列与列之间用管道符号 “|” 隔开，原生方式的表格每一行的两边也要有管道符。
  3. 可在第二行指定不同列单元格内容的对齐方式，默认为左对齐，在 “-” 右边加上 “:” 为右对齐，在 “-” 两侧同时加上 “:” 为居中对齐。
